# Glossary

## Airline & Aviation Terminology

### A

**ADC (Agent Discount Code)**
A code that provides travel agents with discounted fares when making bookings.

**Ancillary**
Additional products or services sold beyond the base airfare, such as baggage, meals, seat selection, or lounge access.

**API (Advance Passenger Information)**
Passenger data transmitted to government authorities before departure for border control purposes.

**APIS (Advance Passenger Information System)**
System for transmitting passenger manifest data to customs and immigration.

**ARINC**
Aeronautical Radio, Incorporated - a major provider of transportation communications and systems engineering.

**Availability**
The number of seats available for sale on a specific flight.

### B

**BCBP (Bar Coded Boarding Pass)**
IATA standard for boarding pass barcodes (2D barcode format).

**Boarding Pass**
A document (physical or electronic) that gives a passenger permission to board an aircraft.

**Booking Class**
A fare category within a cabin class (e.g., Y, B, M for economy).

**BSP (Billing and Settlement Plan)**
IATA's system for financial settlement between travel agents and airlines.

### C

**Cabin Class**
The service class on an aircraft:
- Economy (Y)
- Premium Economy (W)
- Business (C, J)
- First Class (F)

**Check-in**
The process of confirming passenger presence and assigning seats before a flight.

**CIP (Commercially Important Person)**
A passenger designated for special handling due to their value to the airline.

**CRS (Computer Reservation System)**
Legacy term for systems used to manage bookings (now often called PSS).

### D

**DCS (Departure Control System)**
System handling check-in, boarding, and load control for departing flights.

**Denied Boarding**
When a passenger with a confirmed reservation is not allowed to board due to overbooking.

**Domestic Flight**
A flight within the same country.

### E

**E-ticket (Electronic Ticket)**
A digital version of a paper ticket, stored in the airline's reservation system.

**EMD (Electronic Miscellaneous Document)**
Electronic version of a miscellaneous document used for ancillary services.

**ETD (Estimated Time of Departure)**
The planned departure time for a flight.

**ETA (Estimated Time of Arrival)**
The planned arrival time for a flight.

### F

**Fare**
The price charged for air transportation.

**Fare Basis**
A code representing the fare rules and restrictions (e.g., YLEEUS).

**Fare Family**
A group of related fares with similar restrictions (Basic, Standard, Flexible).

**FFP (Frequent Flyer Program)**
A loyalty program rewarding passengers for their travel.

**Flight Number**
Unique identifier for a flight (e.g., AA123, where AA is carrier code, 123 is flight number).

**Flight Segment**
One leg of a journey from origin to destination.

### G

**GDS (Global Distribution System)**
Computer networks used by travel agencies to access airline inventory (e.g., Amadeus, Sabre, Galileo).

**Group Booking**
A reservation for 10 or more passengers traveling together.

### I

**IATA (International Air Transport Association)**
Trade association of the world's airlines.

**IATA Code**
Two-letter airline code (e.g., AA for American Airlines) or three-letter airport code (e.g., JFK for John F. Kennedy Airport).

**ICAO (International Civil Aviation Organization)**
UN agency regulating international aviation.

**ICAO Code**
Four-letter airport code (e.g., KJFK for JFK) or three-letter airline code (e.g., AAL for American Airlines).

**International Flight**
A flight between two different countries.

**Inventory**
The number of seats available for sale on a flight, by booking class.

**Itinerary**
A passenger's complete travel plan, including all flight segments.

### L

**Load Control**
Managing aircraft weight and balance for safe operation.

**Load Factor**
Percentage of available seats filled with passengers.

**Locator (PNR Locator)**
Six-character alphanumeric code identifying a reservation (e.g., ABC123).

**Leg**
One portion of a flight with intermediate stops.

### M

**MCO (Miscellaneous Charges Order)**
A voucher issued by airlines for various services.

**Minimum Connecting Time**
The shortest time allowed to make a connection at an airport.

**Multi-city Booking**
An itinerary with more than one destination.

### N

**No-show**
A passenger who doesn't arrive for their confirmed flight.

**NDC (New Distribution Capability)**
IATA standard for distributing airline content directly to travel agencies.

### O

**One-way**
A journey from point A to point B with no return.

**Open Jaw**
An itinerary where the return origin differs from the outbound destination.

**OTP (On-Time Performance)**
Percentage of flights departing/arriving on schedule.

**Overbooking**
Selling more seats than available to compensate for no-shows.

### P

**Pax**
Aviation abbreviation for "passengers."

**PNR (Passenger Name Record)**
A record in the airline's reservation system containing all booking details.

**POS (Point of Sale)**
The location or channel where a booking is made.

**PSS (Passenger Service System)**
The core system managing reservations, inventory, and departures.

### R

**RASM (Revenue per Available Seat Mile)**
Key airline profitability metric.

**Record Locator**
See PNR Locator.

**Refund**
Money returned to a passenger after cancellation.

**Revenue Management**
The practice of optimizing fare prices to maximize revenue.

**Round-trip**
A journey from point A to point B and back to point A.

**RPM (Revenue Passenger Mile)**
One fare-paying passenger transported one mile.

### S

**Seat Map**
Visual representation of aircraft seating configuration.

**Segment**
One flight in a multi-flight itinerary.

**SITA (Société Internationale de Télécommunications Aéronautiques)**
Provider of IT and telecommunication services to the air transport industry.

**Split PNR**
Dividing one reservation into multiple separate reservations.

**SSR (Special Service Request)**
Codes for special passenger requirements (e.g., WCHR for wheelchair, VGML for vegetarian meal).

**Standby**
Waiting for a seat on a flight without a confirmed reservation.

### T

**Tag (Baggage Tag)**
10-digit number on a baggage claim check.

**Ticket**
Contract of carriage between airline and passenger.

**Ticket Number**
13-digit unique identifier for an e-ticket.

**Through Fare**
A single price for a multi-segment journey.

**Transfer**
Changing from one flight to another (same airline).

**Transit**
Continuing on the same flight after a stop.

### V

**Validating Carrier**
The airline issuing the ticket (may differ from operating carrier).

**VIP (Very Important Person)**
Passenger requiring special handling.

### W

**Waitlist**
List of passengers hoping for seats on a fully booked flight.

**Weight & Balance**
Calculation ensuring aircraft is within safe operating limits.

## Technical Terminology

### A

**API (Application Programming Interface)**
Interface allowing different software systems to communicate.

**API Gateway**
Single entry point for all client requests to backend services.

**Auth Token**
Cryptographic token proving a user's identity.

**Availability Zone (AZ)**
Isolated location within a cloud region.

### C

**Cache**
Temporary storage for frequently accessed data.

**CDN (Content Delivery Network)**
Distributed network of servers delivering content to users.

**CI/CD (Continuous Integration/Continuous Deployment)**
Automated process for building, testing, and deploying code.

**Circuit Breaker**
Pattern preventing cascading failures in distributed systems.

**CORS (Cross-Origin Resource Sharing)**
Mechanism allowing restricted resources to be requested from another domain.

**CRUD (Create, Read, Update, Delete)**
Basic operations for persistent storage.

### D

**DTO (Data Transfer Object)**
Object carrying data between processes.

**Database Migration**
Version-controlled changes to database schema.

**Docker**
Platform for containerizing applications.

**Docker Compose**
Tool for defining multi-container Docker applications.

### E

**Environment Variable**
Dynamic value affecting running processes.

**Event Bus**
Channel for publishing and subscribing to events.

**Event Sourcing**
Storing state changes as a sequence of events.

### H

**HPA (Horizontal Pod Autoscaler)**
Kubernetes feature for automatic scaling.

**Health Check**
Endpoint verifying service is operational.

### I

**IaC (Infrastructure as Code)**
Managing infrastructure through machine-readable files.

**Idempotent**
Operation producing the same result when called multiple times.

### J

**JWT (JSON Web Token)**
Compact, URL-safe token for authentication.

### K

**Kubernetes (K8s)**
Container orchestration platform.

### M

**Microservice**
Small, independent service focused on a specific business capability.

**Middleware**
Software connecting different components or applications.

**MFA (Multi-Factor Authentication)**
Authentication requiring multiple verification methods.

**Monorepo**
Single repository containing multiple projects.

**Message Queue**
Asynchronous communication pattern using queues.

### O

**OAuth**
Open standard for access delegation.

**ORM (Object-Relational Mapping)**
Technique for converting between incompatible type systems.

**Observability**
Ability to understand system internal state from external outputs.

### P

**PCI DSS (Payment Card Industry Data Security Standard)**
Security standard for organizations handling credit cards.

**Pod**
Smallest deployable unit in Kubernetes.

**Prometheus**
Open-source monitoring and alerting toolkit.

### R

**RBAC (Role-Based Access Control)**
Access control based on user roles.

**Redis**
In-memory data structure store used as cache.

**REST (Representational State Transfer)**
Architectural style for web services.

**RabbitMQ**
Open-source message broker.

### S

**Saga Pattern**
Managing distributed transactions across microservices.

**SLA (Service Level Agreement)**
Commitment between service provider and client.

**SLO (Service Level Objective)**
Target value for a service level indicator.

**SLI (Service Level Indicator)**
Quantitative measure of service level.

**SSL/TLS**
Cryptographic protocols for secure communication.

### T

**Throughput**
Number of operations per unit time.

**TOTP (Time-based One-Time Password)**
Algorithm for generating temporary passwords.

**Turborepo**
High-performance build system for monorepos.

### W

**Webhook**
HTTP callback triggered by an event.

**WebSocket**
Protocol for full-duplex communication over TCP.
